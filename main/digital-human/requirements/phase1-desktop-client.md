# Phase 1：桌面端数字人圆形悬浮客户端

## 1. 概述

将现有的 Web 版数字人交互模块（digital-human）改造为**Electron 桌面应用**。第一阶段**不引入 3D 模型渲染**，聚焦于桌面端的基础交互体验：圆形悬浮 UI、音频通讯、唤醒词检测、WebSocket 连接等核心链路。

### 目标

- 提供一个开机自启、常驻后台的桌面应用
- 待机时显示为圆形呼吸光晕（悬浮在桌面最上层）
- 对话时展开为可交互的主窗口（含角色展示和聊天界面）
- 保留所有现有功能：唤醒词检测、语音对话、MCP 工具等

---

## 2. 技术栈

| 层 | 技术 | 用途 |
|----|------|------|
| 桌面壳 | Electron 28+ | 窗口管理、系统托盘、子进程管理 |
| 渲染 | HTML5 Canvas + DOM | 呼吸光晕动画、圆形遮罩 |
| 音频 | Web Audio API | 播放与录制（复用现有代码） |
| Opus 编解码 | libopus (WASM) | Opus 编码/解码（复用现有代码） |
| 网络 | WebSocket | 与 AI 服务端通信（复用现有代码） |
| 唤醒词 | Python sherpa-onnx 子进程 | 本地唤醒词检测（复用现有运行时） |
| 打包 | electron-builder 24+ | 构建 .exe / .dmg 安装包 |

## 5. 功能模块详情

### 5.1 小圆窗（待机态）

**尺寸与样式**：
- 窗口大小：60×60px（可配置）
- 背景：完全透明（`transparent: true`, `frame: false`）
- 置顶：`alwaysOnTop: true`
- 穿透：默认不穿透，鼠标悬停可交互

**呼吸光晕动画**：
- 使用 Canvas 2D 渲染圆形渐变光晕
- 动画参数：缩放 ±5%，透明度 puls 0.6→1.0，周期 3s
- 颜色根据状态变化：

| 状态 | 颜色 | 效果 |
|------|------|------|
| 待机离线 | 灰色 | 弱呼吸，暗光晕 |
| 待机在线 | 青色/蓝色 | 稳定呼吸 |
| 唤醒中 | 橙色/金色 | 快速脉冲（1s 周期） |
| 对话中 | 绿色 | 稳定 + 频谱波动 |
| 错误 | 红色 | 闪烁提示 |

**交互行为**：
- 鼠标拖拽移动窗口（使用 `BrowserWindow.setPosition()`）
- 双击展开主窗口
- 鼠标悬停显示小提示（tooltip-style）
- 右键菜单（系统托盘已有，可以复用）
- 可通过快捷键（如 `Ctrl+Shift+Space`）唤出

### 5.2 主窗口（对话态）

**尺寸与样式**：
- 窗口大小：580×820px（可配置）
- 背景：半透明磨砂毛玻璃效果（`backdrop-filter: blur(20px)`）
- 圆角：border-radius: 20px
- 阴影：外发光 + 内阴影

**角色展示区**（Phase 1 为 2D 方案）：
- 圆形剪影头像（可用 SVG/Canvas 绘制）

- 表情变化：通过颜色/特效变化表达情绪

### 5.4 唤醒词集成

**架构**：
- Electron 主进程管理 Python 子进程（`start.py` 的等价实现）
- 子进程通过 stdout / WebSocket 与主进程通信
- 主进程通过 IPC 将唤醒事件转发到渲染进程

**实现选项**（按推荐优先级排序）：

| 方案 | 说明 | 复杂度 |
|------|------|--------|
| 方案 A（推荐） | Python 子进程，复用现有 `wakeword_runtime` | 低 |
| 方案 B | 使用 `child_process.spawn` 启动 Python 脚本 | 低 |
| 方案 C | Vosk / Porcupine 的 Node.js 绑定替代 | 中 |
| 方案 D | 用 sherpa-onnx 的 C++ API 编写 Node 原生插件 | 高 |

**Phase 1 采用方案 A**：将现有的 `wakeword_runtime` 目录作为 Python 子进程运行，通过 WebSocket bridge 通信。用户在安装桌面应用时需确保 Python 环境（或后续使用 pyinstaller 打包为独立 exe）。

### 5.5 音频与网络通信

**复用清单**（直接拷贝进 Electron 项目）：

| 文件 | 说明 |
|------|------|
| `js/core/audio/opus-codec.js` | Opus 编码器，无需修改 |
| `js/core/audio/player.js` | 音频播放器，适配 window.AudioContext |
| `js/core/audio/recorder.js` | 录音器，适配 getUserMedia 权限 |
| `js/core/audio/stream-context.js` | 流式播放上下文，无需修改 |
| `js/core/network/websocket.js` | WebSocket 消息处理，适配 Electron |
| `js/core/network/wakeword-bridge.js` | 唤醒词桥接，目标地址改为本地 |
| `js/core/network/ota-connector.js` | OTA 连接器，使用 Electron fetch |
| `js/core/mcp/tools.js` | MCP 工具管理，无需修改 |
| `js/config/manager.js` | 配置管理，改用 electron-store 替代 localStorage |
| `js/utils/blocking-queue.js` | 阻塞队列，无需修改 |
| `js/utils/logger.js` | 日志工具，适配 Node.js 文件日志 |

**适配注意事项**：
- `AudioContext` 在 Electron 中一样可用（Chromium 内核）
- `getUserMedia` 需申请麦克风权限（Electron 自动处理）
- `localStorage` 替换为 `electron-store` 或 `JSON 文件`
- WebSocket 连接逻辑不变，目标地址通过配置读取

## 7. 交互细节设计

### 7.1 呼吸光晕 Canvas 动画

```javascript
// 核心动画参数
const config = {
    idle: {
        scale: [0.95, 1.05],     // 缩放范围
        opacity: [0.6, 1.0],     // 透明度范围
        period: 3000,            // 周期 ms
        color: { r: 66, g: 133, b: 244 },  // 蓝色
        glowRadius: [30, 40],    // 光晕半径
    },
    wake: {
        scale: [1.0, 1.2],
        opacity: [0.8, 1.0],
        period: 800,             // 快速脉冲
        color: { r: 255, g: 193, b: 7 },   // 金色
        glowRadius: [35, 50],
    },
    talking: {
        scale: [0.98, 1.02],
        opacity: [0.7, 1.0],
        period: 500,
        color: { r: 76, g: 175, b: 80 },   // 绿色
        glowRadius: [30, 45],
        // 对话状态下叠加音频频谱波动
    }
};
```

动画效果：
1. **径向渐变圆形**：中心亮，边缘渐隐
2. **外发光**：`ctx.shadowBlur` + 动态 shadowColor
3. **呼吸脉冲**：requestAnimationFrame 循环，线性插值
4. **频谱叠加**（对话态）：用 AnalyserNode 数据驱动光晕半径波动

### 7.2 主窗口 2D 角色动画（Phase 1 替代方案）

在没有 3D 模型时，角色展示区可以使用 Canvas 2D 绘制简易角色：

- **嘴部动画**：根据音频能量绘制椭圆开口度
- **眼睛动画**：简单的眨眼定时器
- **头部微动**：整体小幅度旋转/移动
- **情绪指示**：背景色变化 + 小图标（高兴😊、思考🤔等）

这些可以用 100-200 行 Canvas 2D 代码实现，后期替换 3D 模型时不影响接口。

### 7.3 窗口位置记忆

- 小圆窗位置：`BrowserWindow.getPosition()` → 保存在 `electron-store`
- 主窗口位置：同上
- 应用重启后恢复上次位置（屏幕中心作为兜底）
